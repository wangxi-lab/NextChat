可以实现。Goose 的 skill 是按文件系统发现的，核心形态是：

某个技能目录/
  SKILL.md
  其他辅助文件...
SKILL.md 里需要 frontmatter，至少有 name、description。Goose 源码里限制 skill 名称只能用小写字母、数字、-，最长 64。

Goose 会扫描这些位置：

项目级：
<working_dir>/.agents/skills
<working_dir>/.goose/skills
<working_dir>/.claude/skills

全局级：
~/.agents/skills
Goose config dir/skills
~/.claude/skills
~/.config/agents/skills
其中官方源码注释里把 ~/.agents/skills 标成全局用户 skill 的 canonical writable location。参考：
https://github.com/aaif-goose/goose/blob/main/crates/goose/src/skills/mod.rs
https://github.com/aaif-goose/goose/blob/main/crates/goose/src/skills/client.rs

可以在 NextChat 里做三类能力：

安装 skill
在 NextChat 增加“安装 Goose Skill”，把用户选择的目录/zip/Git 地址写入：

全局：~/.agents/skills/<skill-name>/SKILL.md
或项目级：<项目目录>/.agents/skills/<skill-name>/SKILL.md
查询已安装 skill
有两种方式：

直接由 NextChat/Tauri 扫描上述目录里的 SKILL.md
或调用 Goose 后台接口 /config/slash_commands?working_dir=...，它会把 skill 作为 CommandType::Skill 返回。参考：
https://github.com/aaif-goose/goose/blob/main/crates/goose-server/src/routes/config_management.rs
指定 skill 执行
可在“本地Agent”按钮旁加一个 skill 下拉框。用户选中 skill 后，调用 Goose 时把指令包装成类似：

请使用名为 <skill-name> 的 Goose skill 来完成下面的问题。
如果需要，请先加载该 skill 的说明和辅助文件。

用户问题：
...

是的，这个判断对。只把 SKILL.md 内容注入上下文，只适合“纯说明型 skill”。如果 skill 依赖很多 scripts/、assets/、模板文件或相对路径，单纯把 Markdown 塞给 Goose 会丢失 skill 的工作目录语义，所以 Goose 执行时就会找不到这些文件。

更合适的方案是：不要手动注入 SKILL.md 全文，而是把 skill 安装到 Goose 可发现目录，然后让 Goose 自己加载 skill。

Goose 的 skill loader 会知道：

skill 的真实目录路径
SKILL.md
supporting files
用户指定 skill-name/path/to/file 时如何读取辅助文件
相对路径安全边界
落地到 NextChat 可以这样做：

安装 skill
把完整 skill 目录复制到：

全局：~/.agents/skills/<skill-name>/
或项目：<working_dir>/.agents/skills/<skill-name>/
查询 skill
调 Goose 接口：

/config/slash_commands?working_dir=<working_dir>
过滤 command_type === "Skill"
指定 skill 执行
发给 Goose 的 prompt 不注入全文，而是写：

请使用 Goose skill: <skill-name> 来完成下面任务。
如果需要辅助文件，请通过 skill 工具加载该 skill 的 supporting files。

任务：
...
更强一点，NextChat 可以在发问前主动调用 Goose 的 tool：

POST /agent/call_tool
{
  "session_id": "...",
  "name": "load_skill",
  "arguments": {
    "name": "<skill-name>"
  }
}
不过这要求当前 Goose session 的 skills extension 已经加载，并且 load_skill 对 app 可见。需要我们实际测一下你这个 goosed.exe 版本是否允许 /agent/call_tool 调 load_skill。

所以结论是：

SKILL.md 注入：简单，但不支持复杂 skill。
安装完整目录 + Goose load_skill：正确，能支持 scripts/assets/supporting files。
NextChat 要做 skill 管理，应该管理完整目录，而不是只管理 Markdown 内容。
