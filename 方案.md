可以实现。Goose 的 skill 是按文件系统发现的，核心形态是：

某个技能目录/
  SKILL.md
  其他辅助文件...
SKILL.md 里需要 frontmatter，至少有 name、description。Goose 源码里限制 skill 名称只能用小写字母、数字、-，最长 64。

Goose 会扫描这些位置：

项目级：
<working_dir>/.agents/skills
<working_dir>/.goose/skills
<working_dir>/.claude/skills

全局级：
~/.agents/skills
Goose config dir/skills
~/.claude/skills
~/.config/agents/skills
其中官方源码注释里把 ~/.agents/skills 标成全局用户 skill 的 canonical writable location。参考：
https://github.com/aaif-goose/goose/blob/main/crates/goose/src/skills/mod.rs
https://github.com/aaif-goose/goose/blob/main/crates/goose/src/skills/client.rs

可以在 NextChat 里做三类能力：

安装 skill
在 NextChat 增加“安装 Goose Skill”，把用户选择的目录/zip/Git 地址写入：

全局：~/.agents/skills/<skill-name>/SKILL.md
或项目级：<项目目录>/.agents/skills/<skill-name>/SKILL.md
查询已安装 skill
有两种方式：

直接由 NextChat/Tauri 扫描上述目录里的 SKILL.md
或调用 Goose 后台接口 /config/slash_commands?working_dir=...，它会把 skill 作为 CommandType::Skill 返回。参考：
https://github.com/aaif-goose/goose/blob/main/crates/goose-server/src/routes/config_management.rs
指定 skill 执行
可在“本地Agent”按钮旁加一个 skill 下拉框。用户选中 skill 后，调用 Goose 时把指令包装成类似：

请使用名为 <skill-name> 的 Goose skill 来完成下面的问题。
如果需要，请先加载该 skill 的说明和辅助文件。

用户问题：
...
更稳的一版是 NextChat 自己读取该 skill 的 SKILL.md，把内容注入到发给 Goose 的上下文里，这样不依赖 Goose 是否自动触发 skill。

“本地Agent”按钮
skill 选择下拉框
